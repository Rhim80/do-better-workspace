## 스킬 8종 한눈에

명령어는 영문이지만 의미는 한 줄로 외우면 됩니다.

| 명령어 | 한 줄 의미 | 언제 씀 |
|--------|-----------|---------|
| `store-interview` | **내 매장 인터뷰** | "personal-context 어떻게 채우지?" — AI가 단계별로 묻고 자동으로 채워줌 |
| `zone-fetch` | **상권 데이터 모으기** | 12개 공공 데이터 소스에서 한 번에 가져옴 (한 번만, 결과는 캐시) |
| `zone-diagnosis` | **내 자리 진단** | "지금 우리 상권 어떤 상태?" — 매출·유동·외국인·검색 4분면으로 본다 |
| `entry-verdict` | **새 자리 판정** | "이 자리 들어갈까?" — Go / 조건부 / 안됨 한 페이지 결론 |
| `persona-match` | **내 손님 결 맞춤** | "내 손님이 그 동네에 있나?" — 페르소나 비중 + 시간대 매칭 |
| `competitive-pressure` | **경쟁 압력 점수** | "거기 얼마나 빡센가?" — 경쟁 밀도·폐업률·카니발 거리 |
| `concept-fit` | **객단가·콘셉트 적합도** | "내 가격대가 맞는 동네인가?" — 상권 평균 대비 위치 |
| `zone-report` | **한 페이지 카드 출력** | 진단·판정 결과를 PDF로 뽑기 |

**기본 흐름**:
1. **`store-interview`** — 내 매장 인터뷰 (personal-context.md 채우기)
2. `zone-fetch` (1회, 보통 자동 호출)
3. `zone-diagnosis` (지금 자리) 또는 `entry-verdict` (새 자리)
4. 필요 시 보조 3종 (persona-match · competitive-pressure · concept-fit)
5. `zone-report` (PDF로 뽑기)

### Phase 2 — 내 자리 진단 (5-7분, 13:35~13:42)

**입력**:
```
zone-diagnosis 홍대입구    # 내 자리 진단: 지금 홍대 상권 어떤 상태?
```

**무엇이 일어나는가**:
- 12개 공공 데이터 소스에서 홍대 상권 데이터를 끌어옴
- 행정동 코드 매칭 → 캐시 확인 → 병렬 호출
- 시연용은 사전 캐시(빠른 진행), 오후엔 라이브 호출 가능

**출력 카드에서 봐야 할 것**:
- **4분면 신호**: 생활인구 vs 추정매출 — 어느 사분면인가
- **카테고리 변화율**: 카페 vs 인접 5개 (한식·양식·주점·디저트…)
- **외국인 방문 추세**: 회복/감소
- **검색 관심도**: "홍대 카페" 12개월 추세
- **가설 3개**: 데이터에서 도출되는 해석

**해석 가이드**:
- 유동인구 유지인데 카페만 빠진다 → 컨셉·운영 이슈일 가능성
- 인접 카테고리도 같이 빠진다 → 상권 자체가 위축
- 둘 다 오른다 → 정상 성장. 다음 질문은 "임대 상승률은?"


**3곳 한 줄 요약**:
- **한남**: 비소비형 체류 진행 중이지만 페어링 객단가(15,000원) 가정 시 매출 잠재력 충분. 단 임대가 BEP +24%, **협상 여지 있음**

- **압구정**: 프리미엄이지만 임대 너무 비싸 카페 객단가 매출로 회수 어려움 — **임대 과대**

**Verdict 결과**:
- 한남 → **Conditional** (조건부)
- 성수 → **Go**
- 압구정 → **No-Go**

> Verdict는 진실이 아닌 데이터 기반 가설. 마지막 판단은 사람이 한다.

### 라이브 호출 폴백

`zone-fetch` 라이브 호출 실패 시:
- "캐시 4개 중 본인 후보지와 가장 가까운 곳" 폴백
- 참가자에게 "본인 후보지는 점심 후 라이브로 다시" 안내

### 면책 라벨 (시연 중 화면에 노출)

- AI는 자리 결정자가 아닌 의사결정 보조 도구
- BEP 임대는 가이드, 실제 시세는 부동산 플랫폼 호가로 검증
- 추정매출은 모델링 추정치
- Verdict는 데이터 패턴, 최종 판단은 사람

### 라이브 입력 ↔ 캐시 둘 다 운용

- **시연용 (Phase 2~3)**: 사전 캐시 — 흐름 안정
- **오후 (Phase 5 이후)**: 참가자 본인 후보지는 라이브 호출

## 자주 나올 질문 (Q&A)

### Q1. AI가 뽑은 결과 믿어도 되나요?
Verdict는 데이터 패턴이지 진실이 아닙니다. 5-Layer 점수 + Magnitude는 추정·평균치. 실제 가게는 운영 역량·메뉴·마케팅이 더 큰 변수입니다. 본 도구는 직관 + 데이터를 한 자리에 모아 의사결정을 한 단계 깊게 만드는 보조 도구.

### Q2. 추정매출 신뢰도는 어디까지인가요?
서울 우리마을가게 추정매출은 카드사·통신사·공공 데이터 조합한 모델링 추정치. **절대값보다 P25/P50/P75 percentile + 시계열 추세를 봐야 합니다.** 본인 매장 실 매출과 비교하면 본인 상권의 정확도 감각이 잡힙니다.

### Q3. 임대 시세는 정확한가요?
R-One 매크로 분기 통계 + 네이버 부동산 호가 결합. 호가는 호가일 뿐 실제 계약가는 다릅니다. 권리금도 별도. **BEP 임대 ↔ 시세 비교는 의사결정 가이드, 협상 시 본인 발품으로 검증.**

### Q4. 본인 매장 데이터 노출 부담은?
`personal-context.md`에 적은 매장 정보는 본인 디스크에만 저장. AI(Claude)에 입력은 되지만 학습 데이터로 들어가지 않음. 부트캠프 끝난 후 본인 워크스페이스로 가져가서 계속 본인 운영에 사용 가능.

### Q5. 우리 가맹 데이터는 왜 안 들어가나요?
공정위 가맹사업거래 API는 별도 승인이 필요해 부트캠프 시점엔 제외. 부트캠프 후 신청하면 가맹 비중·본사 단위 분석 추가 가능. 지금은 소진공 상가정보로 매장 절대수만 봅니다.

### Q6. 진행자 본점 데이터, 다른 사람도 보고 있는데 괜찮나요?
화면에 보이는 `personal-context-imicoffee.md`는 매장 결만 적은 자료. 매출 일부 구체 숫자는 시연용으로 표준화한 값. 본인 매장 정보는 본인 `personal-context.md`에 적어주세요.
