## Mission 1: 3월 선박 작업 실적 + GMPH 분석

박선호 팀장이 물어본 건 단순한 숫자가 아니다. 한형석 실장은 "왜 이 선석 GMPH가 낮은가?"까지 물어볼 것이다.

운영팀 파일과 장비 마스터를 교차하지 않으면 답이 안 나온다.

> **박선호 팀장**
> 선석별 GMPH 비교해보면 S2가 계속 낮거든요. 장비 노후화 문제인지, 아니면 다른 이유가 있는지 판단이 안 서요.
> 기상악화 기간 데이터도 같이 보면 좋을 것 같은데.

### 분석 요청

```
@bpt_vessel_operations_2603.csv @bpt_equipment_master.csv

3월 선박 작업 실적 분석을 해줘.

1. 선석별(S1~S5, G1~G3) GMPH 평균을 계산하고, 전체 평균 대비 상하위 선석을 구분해줘
2. 지연 사유 분포를 보여줘 - 특히 3월 12~19일 기상악화 기간 전후 비교
3. ARMGC와 STS 장비 중 MTBF(평균 고장 간격)가 300시간 미만인 장비 목록과,
   해당 장비가 배치된 선석의 GMPH를 교차해서 장비 노후화가 실적에 미치는 영향을 분석해줘
4. 크레인 투입 대수(sts_cranes_assigned)와 GMPH의 상관관계도 정리해줘
```

### 결과 확인 후 추가 질문

```
GMPH 하위 선석 3개를 골라서, 각 선석의 지연 사유 Top 3와
해당 선석에 배치된 장비의 평균 연식(year_installed)을 함께 보여줘.
"장비 노후화 vs 운영 요인"으로 분류해줄 수 있어?
```

```
3월 기상악화 기간(3/12~3/19) 동안 delay_reason이 "기상악화"인 콜 목록을 뽑고,
그 기간 GMPH가 평소 대비 몇 % 저하됐는지 계산해줘.
주주사 보고용 한 줄 요약도 만들어줘.
```

### 도전 과제

```
선석별 GMPH 분석 결과를 HTML 대시보드로 만들어줘.
선석별 막대 차트, 지연 사유 파이차트, 장비 노후도 색상 구분(10년 이상=빨강, 5~10년=노랑, 5년 미만=초록).
파일명 bpt_mission1_dashboard.html로 저장해줘.
```

## Mission 2: 선사별 물동량 + 환적 비중 분석

이정민 과장의 요청. HMM 계약 갱신은 연간 매출의 35%가 걸린 협상이다.

"HMM이 3월에 얼마나 했는지"가 아니라 "HMM이 우리 터미널에 왜 머물러야 하는지"를 설명해야 한다.

> **이정민 과장**
> HMM 담당자가 계약 갱신 전에 "다른 터미널도 알아보고 있다"고 했어요.
> 우리 터미널 환적 처리 속도가 경쟁력이라는 걸 데이터로 보여줄 수 있으면 좋겠어요.
> 환적(T/S) 비율이랑 선사별 GMPH 비교가 핵심일 것 같아요.

### 분석 요청

```
@bpt_vessel_operations_2603.csv

선사별 물동량과 환적 비중 분석을 해줘.

1. 선사별 총 물동량(total_moves) 순위와 환적 비율(ts_teu / total_moves) 비교
2. HMM 상세 분석:
   - 3월 총 콜 수, 평균 GMPH, 평균 지연 시간
   - 환적 비율이 다른 선사 대비 높은지 낮은지
   - 냉동 컨테이너(reefer_count) 비중
3. 선사별 GMPH 분포(최소, 평균, 최대)를 표로 정리해줘
4. 물동량 상위 5개 선사의 월별 흐름이 3월 내에서 어떤지 (상반월/하반월 분리)
```

### 결과 확인 후 추가 질문

```
HMM vs MSK(Maersk) GMPH 직접 비교를 해줘.
GMPH 차이가 선사 요인(선사 지연)인지 터미널 요인(장비, 기상)인지 delay_reason 기준으로 분류해줘.
HMM 계약 갱신 협상용 1페이지 요약 포인트 3개 만들어줘.
```

```
환적 비율이 50% 이상인 선박 콜 목록을 뽑아줘.
그 콜들의 평균 GMPH와 전체 평균을 비교하고,
"환적 특화 선석" 운영 가능성에 대한 의견도 달아줘.
```

### 도전 과제

```
선사별 물동량 + GMPH 비교 대시보드를 HTML로 만들어줘.
가로 막대차트(선사별 물동량), 버블 차트(물동량 vs GMPH vs 환적비율).
HMM은 강조 색상으로 표시해줘.
파일명 bpt_mission2_dashboard.html로 저장해줘.
```

## Mission 3: 야드 혼잡도 + 냉동 플러그 현황

성수기 대비가 필요한 건 선박 측뿐 아니다. 야드가 막히면 배를 받을 수 없다.

> **박선호 팀장**
> 야드 쪽 좀 봐주세요. R블록 냉동 플러그가 거의 찬 날이 많았거든요.
> 4월에 냉동 물량이 더 들어오면 어떡합니까.
> 온도 이상(temp_alarm) 나온 날이랑 플러그 점유율 높은 날이 겹치는지도 보고 싶어요.

### 분석 요청

```
@bpt_yard_daily_2603.csv

3월 야드 혼잡도와 냉동 플러그 현황 분석을 해줘.

1. 전체 블록의 일별 평균 점유율(current_teu / capacity_teu) 추이
   - 점유율 85% 초과 일수가 며칠인지
   - 신선대 vs 감만 비교
2. R블록(냉동 전용) 분석:
   - 플러그 점유율(reefer_plugged / reefer_capacity) 90% 초과 일수
   - 온도 이상 알림(temp_alarm_count) 발생 날짜 목록
   - 플러그 점유율과 온도 알림 상관관계
3. 장치료 초과 컨테이너(containers_over_freetime) 일별 추이
4. gate_in - gate_out 순 반입(net inflow) 추이: 재고 축적 트렌드
```

### 결과 확인 후 추가 질문

```
R블록 5개 중 플러그 점유율이 가장 높은 블록을 찾아줘.
3월 31일 기준 현황과 4월 성수기 물량 10% 증가 시 용량 부족 예상일 수도 계산해줘.
```

```
장치료 초과 컨테이너가 많은 블록 Top 5를 뽑고,
해당 블록의 avg_dwell_days와 gate_out 수치를 같이 보여줘.
"장치 효율 저하" 원인 분석 한 줄 요약도 달아줘.
```

## Mission 4: 선사별 청구 + 미수금 관리

최수진 대리의 문제는 숫자가 어디서 틀렸는지 모른다는 것이다. 그리고 미수금은 갈수록 쌓이고 있다.

> **최수진 대리**
> MSK가 "우리 실제 반입 물량이랑 청구 TEU가 다르다"고 하는데,
> vessel_operations 파일이랑 billing 파일을 같이 봐야 하는데 제가 엑셀로 하면 너무 복잡해요.
> 미수금 현황이랑 dispute 패턴도 한눈에 보고 싶어요.

### 분석 요청

```
@bpt_billing_summary_2603.csv @bpt_vessel_operations_2603.csv

선사별 청구 현황과 미수금 분석을 해줘.

1. 선사별 청구 총액(total_amount) 순위 - 하역료/장치료/냉동료/위험물 처리료 구성 비중
2. dispute_flag = Y인 건 목록과 dispute_reason 분류
3. 미수금 현황:
   - 미입금 / 부분입금 건의 총 미수 금액 추산
   - payment_due_date 기준 연체 여부
4. MSK 검증:
   - billing의 teu_volume과 vessel_operations의 MSK total_moves 합계를 비교해줘
   - 차이가 있다면 어느 건에서 발생했는지 추적
```

### 결과 확인 후 추가 질문

```
dispute 이의제기 4건 각각에 대해 "어떤 데이터로 대응할 수 있는가" 정리해줘.
vessel_operations 데이터를 근거로 사용해줘.
```

```
payment_status별(입금완료/부분입금/미입금) 선사 목록과
각 선사의 미수 추산 금액을 보여줘.
"4월 말까지 회수 우선순위" 순서도 제안해줘.
```

## Mission 5: 장비 예방정비 + 4월 가동 계획

4월은 성수기다. 장비가 멈추면 선박 운항 일정이 밀린다. 정비 일정과 성수기 물량이 충돌하지 않아야 한다.

> **박선호 팀장**
> STS-02 정비가 4월 중으로 잡혀 있는데, S2 선석 운영이 빡빡해질 것 같아요.
> MTBF 낮은 장비들 먼저 정비 해주고, 4월에 문제 생길 것 같은 장비 목록 좀 뽑아주실 수 있어요?

### 분석 요청

```
@bpt_equipment_master.csv

4월 장비 예방정비 계획 분석을 해줘.

1. next_pm_date가 2026-04-01 ~ 2026-04-30 사이인 장비 목록
   - equipment_type, terminal, 현재 maintenance_status 포함
   - STS Crane 우선 정렬
2. MTBF가 200시간 미만인 장비 목록 - "고위험 장비" 분류
3. 4월 정비 예정 장비 중 same terminal에 같은 타입 백업 장비가 몇 대인지
   (예: S2 선석 STS 정비 시 대체 가능한 STS 수)
4. year_installed 기준 2010년 이전 장비(16년 이상 노후 장비) 현황
```

### 결과 확인 후 추가 질문

```
4월 정비 예정 STS Crane 목록에서, 같은 terminal의 나머지 STS 대수를 계산해줘.
"정비 중 선석 운영 가능 여부" 판단 기준을 크레인 최소 2대로 설정했을 때,
운영 불가 위험 기간이 있는지 표로 보여줘.
```

```
MTBF 낮은 순으로 상위 10개 장비를 뽑고,
last_pm_date 기준 last PM 후 경과 일수를 계산해줘.
"즉시 정비 필요" vs "30일 이내 정비 권고" vs "정상" 분류해줘.
```

## 항만 용어 설명

- **GMPH (Gross Moves Per Hour)**: 크레인 1대가 시간당 처리하는 컨테이너 이동 횟수. 터미널 생산성의 핵심 지표
- **STS Crane (Ship-to-Shore Crane)**: 선박에서 야드로 컨테이너를 내리는 안벽 크레인
- **ARMGC (Automated Rail-Mounted Gantry Crane)**: 야드 내 컨테이너를 자동으로 정렬하는 갠트리 크레인
- **T/S (Transshipment, 환적)**: 한 선박에서 내린 컨테이너를 다른 선박에 다시 싣는 것. 부산항 물동량의 핵심
- **TEU (Twenty-foot Equivalent Unit)**: 20피트 컨테이너 1개 단위. 항만 물동량 측정 기준
- **Freetime (장치 무료 기간)**: 컨테이너를 터미널에 무료로 보관할 수 있는 기간. 초과 시 장치료(Demurrage) 발생
- **MTBF (Mean Time Between Failures)**: 장비의 고장 간격 평균 시간. 낮을수록 자주 고장남
- **LOA (Length Overall)**: 선박 전장. 선석 배정의 기준이 됨
