## 상황 설정 — 청담동 본점, 4월 마지막 주

청기와옥은 청담동 본점 한 곳을 운영하는 한식당이다. 한정식·등갈비찜·보쌈·갈비탕이 주력이고, 저녁엔 전통주 페어링이 받쳐준다. 작년부터 배민·쿠팡이츠도 시작했고, 4월부터는 쿠팡이츠 광고도 돌리기 시작했다.

월요일 아침 9시. 사장은 오피스에 앉아 어제(일요일) POS 마감지와 배민·쿠팡이츠 사장님광장을 번갈아 띄워보고 있다. 그리고 단톡방엔 이미 알림이 쏟아진다.

> **운영팀장**: "3월보다 4월 매출은 분명 늘었어요. 근데 통장 잔고는 비슷한 느낌이에요. 왜 그런지 한 번 봐주실 수 있을까요?"
>
> **점장**: "지난주에 배달이 갑자기 또 많이 들어왔거든요. 주방이 한참 밀렸어요. 배달이 진짜 남는 장사 맞아요?"
>
> **마케팅 담당 (외주)**: "쿠팡이츠 광고 4월부터 시작했는데, 이게 효과가 있는 건지 평가 좀 해주세요. 5월에도 계속 돌릴까요?"

**(속으로)** *셋 다 결국 같은 질문이다. "4월에 무슨 일이 있었고, 진짜 남는 장사는 어디인가." 매번 사장 직원이 엑셀로 채널별·메뉴별 따로 뽑고, 수수료 계산하고, 일별 그래프 그리고… 이렇게 하면 늘 4시간이다. 그런데 결국 한 번 보고 끝, 다음 달 또 처음부터.*

**Claude Code가 없었다면**: POS 다운로드 → 배민 정산서 다운로드 → 쿠팡이츠 정산서 다운로드 → 엑셀 4개 시트 합치기 → 메뉴별 원가 매핑 → 채널별 수수료 차감 → 차트 만들기 → 주말까지 8시간+
**Claude Code가 있다면**: 1시간 안에 답 + 매일 아침 자동 보고서까지.

## Mission 1 — 4월 채널·메뉴 수익성 진단 (3시간 → 30분)

**누가 이걸 묻고 있나**: 운영팀장 + 점장. "매출은 올랐는데 왜 마진은 안 늘지?"

### 분석 요청

```
@50-resources/sample-data/nanro/menu_master.csv
@50-resources/sample-data/nanro/pos_sales_2603.csv
@50-resources/sample-data/nanro/pos_sales_2604.csv
@50-resources/sample-data/nanro/delivery_sales_2603.csv
@50-resources/sample-data/nanro/delivery_sales_2604.csv

청기와옥 본점의 3월 vs 4월 채널별·메뉴별 수익성을 비교해줘.

반드시 봐야 할 관점:
1. 매출은 늘었는데 정산순매출(수수료/배달비 차감 후)은 얼마나 늘었는가
2. 배달 비중이 늘면서 실제 마진이 어떻게 변했는가 (배민 vs 쿠팡이츠 분리)
3. 메뉴별 기여 마진 TOP 10 / BOTTOM 10 (홀 가격과 배달 가격이 다르므로 채널별로)
4. 4월 쿠팡이츠 광고비 효과는 — 광고비 회수 됐는지

뉴믹스 sales-report-2602.md 형식과 동일한 톤·구조로 작성해줘.
```

### 결과 확인 후 후속 질문

```
배달 1건당 평균 정산순매출과 평균 마진을 홀 1건과 비교해줘.
배달 1건이 홀 1건보다 마진이 적다면, 그 차이를 광고/수수료/배달비로 분해해서 보여줘.
```

```
배달 메뉴 중에 "팔수록 손해 보는" 메뉴가 있는지 찾아줘.
판단 기준: 배달가 - 원가 - (배달가 × (앱+PG+광고수수료율)) - 사장배달비 < 0 이면 적자.
객수가 많은 적자 메뉴는 배달 가격을 얼마로 올리면 BEP인지 함께 제안해줘.
```

```
요일별 채널별 매출 패턴을 보여줘.
주방이 밀리는 금/토 저녁에 배달을 일시적으로 OFF 했을 때 잃을 매출을 추정해주고,
그 시간대에 홀에 더 받을 수 있는 손님 매출과 비교해줘.
```

### 도전 과제 — 한 페이지 대시보드

```
위 분석을 한 페이지 HTML 대시보드로 만들어줘.
- 상단: 3월 vs 4월 핵심 4개 숫자 (총매출 / 정산순매출 / 마진 / 배달비중)
- 좌측: 채널별 수익성 비교 막대그래프
- 우측: 메뉴 기여 마진 TOP 10 / BOTTOM 10
- 하단: 요일·채널 히트맵
- 모바일에서도 보이게 반응형
파일은 50-resources/sample-data/nanro/profitability-dashboard.html 에 저장.
```

## Mission 2 — 매일 아침 받아보는 1장짜리 모닝 브리프 (매일 30분 → 매일 0분)

**누가 이걸 묻고 있나**: 사장 본인. "매일 아침 정산서 띄워서 비교하는 거 그만하고 싶다."

### 분석 요청

```
@50-resources/sample-data/nanro/daily_summary.csv

오늘이 2026-04-30 이라고 가정하고, 어제(2026-04-29 수)의 마감 데이터를 기준으로
사장이 출근 전 5분 안에 읽을 수 있는 "1장짜리 모닝 브리프" 보고서를 만들어줘.

브리프에 들어가야 할 것:
1. 어제 한 줄 요약 — 총매출 / 정산순매출 / 추정마진 / 전주 같은 요일 대비 증감
2. 채널별 어제 숫자 (홀 / 배민 / 쿠팡이츠) + 각 채널 7일 이동평균과 어제 비교
3. 어제 특이점 3개 (좋았던 것 / 안 좋았던 것 / 모니터링 필요)
4. 오늘 액션 제안 3개 (재고/예약/광고 관점에서 구체적으로)

톤: 짧고, 사장이 바로 결정 가능한 형태. 표 + 짧은 문장.
형식은 마크다운, 50-resources/sample-data/nanro/morning-brief-2604-30.md 에 저장.
```

### 결과 확인 후 후속 질문

```
이 브리프를 매일 아침 8시에 같은 형식으로 받고 싶어.
어제 날짜만 바뀌면 동일하게 동작하도록, 실행 명령어 한 줄로 만들 수 있는 워크플로우를 설계해줘.
(스킬로 만들 거라면 SKILL.md 골격까지 같이 보여줘.)
```

```
주간(월~일) 단위 브리프도 같은 톤으로 만들어줘.
요일별 매출 + 주력 메뉴 TOP 5 + 배달 수수료 비중 + 다음 주 예약 가설.
파일은 weekly-brief-2604-W4.md 형태로.
```

### 도전 과제 — Slack/카톡 자동 발송

```
위 모닝 브리프를 매일 아침 8시에 사장 핸드폰으로 받고 싶어.
가능한 발송 채널 옵션 3개를 비교해줘 (Slack / 카카오워크 / Telegram bot).
각각 설정 난이도, 비용, 가독성, 자동화 안정성을 표로.
가장 실용적인 옵션을 골라서, 실제로 매일 아침 보낼 수 있는 코드 골격까지 작성해줘.
```

## 더 파고들기 — 두 미션 결과 연결

```
Mission 1에서 나온 "배달은 마진이 낮다"는 결론과, Mission 2 모닝 브리프의 일별 흐름을 합쳐서:
- 4월 한 달 동안 배달 비중이 가장 높았던 날 TOP 5와 그날의 마진 추이를 보여줘.
- "배달이 많은 날은 마진이 낮다"가 데이터로 증명되는지 확인.
- 만약 그렇다면, 사장이 "배달 비중을 어디까지만 받을지" 일별 상한선을 어떻게 정할 수 있을지 제안.
```

```
3~4월 데이터로 "어떤 요일 + 어떤 시간대 + 어떤 메뉴 조합"이 가장 마진이 좋은지 찾아줘.
그리고 그 조합을 더 끌어올릴 수 있는 5월 운영 액션 3개를 제안해줘.
(예: 메뉴 진열 순서, 직원 배치, 예약 마감 시간 등)
```

```
청기와옥 본점 한 곳 데이터만 봤는데, 만약 동일 컨셉으로 2호점을 낸다면 어느 동네가 좋을지
부트캠프에서 배운 entry-verdict 스킬과 어떻게 연결할 수 있을지 흐름을 설명해줘.
(이 분석 결과 + entry-verdict 입력을 어떻게 이어붙일까)
```

오후 3시. 사장은 단톡방에 dashboard 링크와 모닝 브리프 샘플을 던진다.

> **사장**: "오늘부터 매일 아침 8시에 자동으로 받아볼게요."
>
> **운영팀장**: "이거… 제가 원하던 거예요. 매일 정산서 6개 띄워서 비교하던 거 끝났네요."
>
> **점장**: "배달 적자 메뉴 3개 진짜 있네요. 오늘 배민 사장님광장에서 가격 조정하겠습니다."
>
> **마케팅 외주**: "쿠팡이츠 광고비 4월에 회수 못 했으면, 5월엔 시간대를 점심에만 한정해서 다시 테스트해볼게요. 다음 달 데이터 같이 봐요."

**(속으로)** *내가 직접 엑셀 안 돌려도 답이 나온다는 게 이런 거구나. 이게 가능하면, 매장 2호점 들어갈지 말지도 같은 결로 의사결정 할 수 있을 것 같다. 난로회에서 배운 entry-verdict 스킬, 우리 데이터 + 입지 데이터 결합하면 진짜로 한 페이지에서 결정이 나올 듯. 다음 출점 검토 때 한 번 돌려보자.*

**시즌/이벤트 (분석 시 참고)**:
- 3/1~3/2 삼일절 (휴장 또는 매출 -45%)

- 4/25 폭우 (배달 수요 변화)
- 4/30 부처님오신날
- 정기휴무: 매월 첫째·셋째 월요일
