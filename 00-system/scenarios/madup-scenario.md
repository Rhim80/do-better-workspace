## 상황 설정

**월요일 오전 9시. 서초 사무실.**

담당 클라이언트 6개사. 오늘 해야 할 일:

1. 글로우랩 3월 월간 리포트 — LEVER에서 데이터는 뽑았다. 대시보드도 있다. 문제는 **클라이언트에게 보낼 리포트 텍스트**를 써야 한다는 것. "ROAS 449%입니다"로 끝나면 클라이언트가 묻는다: "그래서요?" 화이트데이 효과, 매체별 예산 재배분 방향, 4월 전략까지 써야 한다. 클라이언트당 2시간, 6개사면 이틀.

2. 신규 클라이언트 경쟁PT 제안서 — 뷰티 브랜드 '블룸랩' RFP가 들어왔다. 에코마케팅, 와이즈버즈와 경쟁. 제안서를 2주 안에 내야 하는데, 낮에는 기존 클라이언트 실무를 봐야 하니까 사실상 저녁/주말 작업이다. 시장 조사, 경쟁사 분석, 미디어 전략, PPT 구조까지.

3. 글로우랩 4월 소재 교체 — 3월 소재 성과 데이터를 보고 4월 방향을 잡아서 디자이너에게 크리에이티브 브리프를 넘겨야 한다. 수정 요청이 평균 90%라서, 처음에 방향을 명확하게 잡지 않으면 AE-디자이너 사이에 핑퐁이 한 주 내내 이어진다.

슬랙을 연다.

> **기유진 이사**: 글로우랩 월간 리포트 수요일까지 부탁해요
> 화이트데이 선물세트 캠페인 효과를 보고 싶고
> 4월 매체 예산 재배분 방향도 넣어주세요
>
> 블룸랩 제안서 진행 상황은 어때요?
> 이번엔 에코마케팅이랑 와이즈버즈도 들어온다고 하니까
> LEVER Xpert 기반 차별화 포인트 확실하게 잡아주세요

> **최다은 MP**: 글로우랩 4월 소재 교체 건
> 3월 데이터 기반으로 브리프 빨리 넘겨주면
> 디자이너 일정 잡을 수 있어요

**(속으로)**: 리포트 텍스트, 제안서, 브리프... 숫자는 LEVER에 다 있는데, 결국 **사람이 글을 써야 하는 일**이 전부다. 이게 매일 쌓인다.

## Mission 1: 클라이언트 월간 리포트 — "숫자 뒤의 이야기"

LEVER가 해주는 것: 매체별 노출/클릭/전환/ROAS 숫자, 대시보드, 차트.
AE가 해야 하는 것: **"왜 이런 결과가 나왔는지"**, **"그래서 다음 달에 뭘 해야 하는지"**.

이 텍스트 작성이 클라이언트당 2시간. 6개사면 12시간. 매달 반복.

> 사용 데이터: `madup_campaign_daily_2603.csv` (3,472건) + `madup_client_master.csv` (28건)

### 분석 요청

```
"글로우랩의 3월 LEVER 데이터를 분석해서,
클라이언트 담당자에게 보낼 월간 리포트를 작성해줘.

포함할 내용:
1. 3월 성과 요약 (매체별 ROAS, 총 광고비, 전환매출)
2. 주요 발견 3가지 (데이터에서 눈에 띄는 패턴)
3. 화이트데이(3/14) 캠페인 효과 분석
4. 4월 전략 제안 (매체별 예산 재배분 방향)

톤: 전문적이지만 읽기 쉽게.
분량: A4 2페이지 이내."
```

### 후속 질문

```
"핀크레딧은 KPI가 CPA 25,000원인데, 3월에 달성했어?
달성 여부와 원인, 4월 개선 방향을 핀크레딧 담당자에게 보낼 형태로 써줘"
```

```
"6개 클라이언트 전체의 3월 성과를 한눈에 볼 수 있는
내부 보고용 1페이지 요약을 만들어줘.
클라이언트별 KPI 달성 여부, 수수료 수익, 리스크 항목 포함"
```

### 이 미션의 핵심

"데이터 분석"이 아니다. LEVER 데이터에 **맥락과 판단을 입혀서 글로 써주는 것**이다. 같은 ROAS 380%라도, 목표가 400%인 클라이언트에겐 "미달 + 원인 + 개선안"이고, 목표가 300%인 클라이언트에겐 "초과 달성 + 성공 요인 + 확대 제안"이다. 이 판단을 매번 AE가 직접 쓰고 있었다.

## Mission 2: 경쟁PT 제안서 — "저녁마다 쓰던 제안서"

에이전시 AE의 가장 큰 시간 블랙홀. 2주~1개월, 낮에 실무 + 저녁에 제안서 병행. PPT 1장 수정에 4시간. 수주 실패하면 모든 작업이 소멸.

> 사용 데이터: `madup_competitor_benchmark.csv` (8건) + 웹 리서치

### 상황

뷰티 브랜드 '블룸랩'(클린뷰티, 20대 후반~30대 타겟)이 RFP를 보내왔다. 기존 에이전시(A사) ROAS 280%로 부진, 교체 검토 중. 에코마케팅, 와이즈버즈와 경쟁PT.

**RFP 요약**:
- 목표: 자사몰 ROAS 400% 이상
- 월 광고비: 1.5억
- 매체: Meta + Naver SA + TikTok 중심
- 기간: 6개월 (2026년 5월~10월)

### 단계별 요청

**Step 1: 시장 리서치**
```
"블룸랩이 속한 클린뷰티 시장을 조사해줘.
한국 시장 규모, 성장률, 주요 브랜드, 최근 트렌드.
제안서 '시장 환경' 섹션에 넣을 수 있는 형태로"
```

**Step 2: 경쟁 에이전시 분석**
```
"competitor_benchmark 데이터와 웹 리서치를 합쳐서,
에코마케팅과 와이즈버즈 대비 매드업의 차별화 포인트를 정리해줘.
매드업만의 강점: LEVER Xpert 기반 데이터 운영 + 자체 AI 솔루션"
```

**Step 3: 미디어 전략**
```
"월 1.5억 예산으로 Meta + Naver SA + TikTok 미디어 믹스를 설계해줘.
매체별 예산 배분, 예상 KPI(노출/클릭/전환/ROAS), 배분 근거 포함.
기존 에이전시가 ROAS 280%밖에 못 낸 원인 추정과 개선 방향도"
```

**Step 4: 제안서 구조**
```
"위 리서치를 종합해서 경쟁PT 제안서 전체 구조를 만들어줘.

목차 + 각 섹션별 핵심 메시지 + 슬라이드 키 카피.
PPT로 바로 옮길 수 있는 수준으로.
총 15~20장 분량."
```

### 이 미션의 핵심

제안서 작성에서 가장 시간이 걸리는 건 "자료 조사"와 "구조 잡기"다. 아이디어가 나오면 PPT 작업은 비교적 빠르다. Claude Code가 리서치 → 분석 → 구조 → 초안 텍스트까지 한 번에 만들어주면, AE는 **아이디어 다듬기와 클라이언트 맞춤 스토리텔링에 집중**할 수 있다. 2주가 반나절로.

## Mission 3: 크리에이티브 브리프 — "수정 90%를 줄이는 방법"

AE가 디자이너에게 넘기는 소재 제작 요청서. 문제는 광고주 수정 요청이 90%. 처음에 방향을 명확하게 잡지 않으면 AE↔디자이너 핑퐁이 일주일 내내 이어진다.

> 사용 데이터: `madup_creative_performance_2603.csv` (101건)

### 요청

**Step 1: 데이터 기반 방향 설정**
```
"글로우랩의 3월 크리에이티브 데이터를 분석해줘.
소재 유형별(image/video/carousel),
콘셉트별(UGC/전문가추천/시즌프로모/혜택강조 등)
CTR과 CVR을 비교해서 어떤 조합이 가장 효과적이었는지 정리"
```

**Step 2: 브리프 생성**
```
"분석 결과를 바탕으로 4월 글로우랩 크리에이티브 브리프를 작성해줘.

포함 항목:
- 추천 소재 유형 + 근거 (3월 데이터 기반)
- 각 소재별 카피 방향 3안
- 사이즈/포맷 스펙
- 제작 수량
- 레퍼런스 설명
- 제작 일정 가이드

디자이너가 '이거 보고 바로 작업 시작할 수 있다'는 수준으로."
```

**Step 3: 카피 시안**
```
"브리프에 맞는 헤드라인 카피를 소재 유형별로 5개씩 써줘.
봄 시즌 + 뷰티 세럼 맥락.
톤: 20대 후반~30대 여성, 성분 관심, 과장 없이 담백하게"
```

### 이 미션의 핵심

크리에이티브 브리프를 "내 감으로" 쓰면 디자이너와의 소통에서 충돌이 생긴다. **"이 소재 유형이 CTR 2배 나왔으니 이 방향으로"**라는 데이터 근거가 있으면, 광고주 수정 요청에도 "데이터 기반 판단입니다"라고 방어할 수 있다. 주관 → 근거.
