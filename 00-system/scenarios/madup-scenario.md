# AX 실습 시나리오 - 매드업 (MADUP)

> **선행 조건**: `claude-code-practice-guide.md`의 Part 1-2 완료
> **샘플 데이터**: `50-resources/sample-data/madup/`

---

## 이 시나리오의 핵심

"AI가 리포트를 써준다"가 아니다. 그건 ChatGPT에 복붙해도 된다.

핵심은 **"AE가 자기 업무 시스템을 직접 만든다"**는 것이다.

- CLAUDE.md에 클라이언트 맥락을 쌓으면, 매번 같은 설명을 반복할 필요가 없다
- 스킬을 한 번 만들면, 다음 달에도 같은 명령어로 같은 퀄리티가 나온다
- Slack, Notion, 메일이 연결되면, "분석 → 공유 → 기록"이 한 흐름이 된다
- 이 모든 걸 개발팀에 요청하지 않고, AE 본인이 직접 만든다

GPT나 Gemini로는 안 되는 이유: 매번 처음부터 시작해야 하고, 파일을 직접 못 읽고, 외부 도구와 연결이 안 되고, 맥락이 휘발된다.

---

## 상황 설정

**월요일 오전 9시. 서초 사무실.**

담당 클라이언트 6개사. 매달 반복되는 일:

- LEVER에서 데이터 추출 → **리포트 인사이트 텍스트 작성** → 클라이언트에게 발송
- 클라이언트당 2시간 × 6개사 = 매달 12시간
- 매번 GPT에 데이터를 복붙하고, 프롬프트를 다시 쓰고, 결과를 복사해서 PPT에 붙이고...

오늘은 이 루틴을 **시스템으로 만드는 날**이다.

---

## Phase 1: 워크스페이스에 "나"를 심는다

> GPT/Gemini와의 결정적 차이: **맥락이 축적된다**

### Step 1: CLAUDE.md에 내 역할과 클라이언트 맥락 등록

```
"CLAUDE.md를 편집해줘. 아래 내용을 추가:

## 내 역할
- 매드업 마케팅본부 AE
- 담당 클라이언트: 글로우랩(뷰티), 어반핏(패션), 프레시밀(F&B),
  펫프렌즈(반려동물), 핀크레딧(금융), 딜리버리킹(O2O)
- 주요 업무: 월간 리포트, 제안서, 크리에이티브 브리프, 클라이언트 커뮤니케이션

## 클라이언트 컨텍스트

### 글로우랩 (뷰티/화장품)
- KPI: ROAS 400%
- 월 광고비: 1.8억 (Meta 28%, Naver SA 30%, GFA 10%, Kakao 12%, TikTok 8%)
- 수수료율: 13%
- 담당자 성향: 데이터 중심, 숫자보다 '그래서 뭘 해야 하는지'를 원함
- 리포트 톤: 간결, 액션 중심, 장황한 설명 싫어함
- 현재 이슈: 화이트데이 캠페인 효과 검증 필요

### 핀크레딧 (금융/핀테크)
- KPI: CPA 25,000원
- 월 광고비: 3억
- 수수료율: 10%
- 담당자 성향: 보수적, 리스크 먼저 봄
- 리포트 톤: 정량적, 목표 대비 실적 명확하게
- 현재 이슈: CPA 목표 미달 시 예산 삭감 가능성"
```

### 이 단계의 의미

이제부터 "글로우랩 리포트 써줘"라고만 하면, Claude Code가 이미 알고 있다:
- KPI가 ROAS 400%라는 것
- 담당자가 액션 중심을 좋아한다는 것
- 톤이 간결해야 한다는 것

GPT에서는? 매번 "이 클라이언트는 뷰티 브랜드이고, KPI는 ROAS 400%이고, 담당자는 데이터 중심이고..."를 다시 써야 한다.

---

## Phase 2: 반복 업무를 스킬로 만든다

> GPT/Gemini와의 결정적 차이: **한 번 만들면 매달 재사용**

### Step 2: /monthly-report 스킬 만들기

```
"매달 반복하는 클라이언트 월간 리포트 작성을 스킬로 만들어줘.

스킬 이름: /monthly-report
입력: 클라이언트명
동작:
1. sample-data에서 해당 클라이언트의 캠페인 데이터를 읽는다
2. client_master에서 KPI 목표와 계약 조건을 가져온다
3. CLAUDE.md의 클라이언트 컨텍스트(담당자 성향, 리포트 톤)를 참조한다
4. 아래 구조로 리포트 인사이트를 생성한다:
   - 3월 성과 요약 (매체별 핵심 지표)
   - KPI 달성 여부 + 원인
   - 주요 발견 3가지 (데이터에서 눈에 띄는 패턴)
   - 다음 달 전략 제안 (구체적 액션 아이템)
5. CLAUDE.md의 톤 가이드에 맞춰 텍스트를 조정한다"
```

### Step 3: 스킬 실행 — 글로우랩

```
/monthly-report 글로우랩
```

결과를 확인하고 다듬는다. 핵심은 **같은 명령어로 다음 달에도 같은 포맷이 나온다**는 것.

### Step 4: 스킬 실행 — 핀크레딧

```
/monthly-report 핀크레딧
```

같은 스킬인데 CLAUDE.md의 클라이언트 컨텍스트가 다르니까:
- 글로우랩은 "간결, 액션 중심"
- 핀크레딧은 "정량적, 목표 대비 실적 명확하게"

**톤과 구조가 자동으로 달라진다.** 스킬은 하나, 맥락은 클라이언트별.

### 이 단계의 의미

GPT에서는 매달 새 대화를 열고, 데이터를 복붙하고, 프롬프트를 다시 쓴다. 여기서는 `/monthly-report 글로우랩` 한 줄이면 끝. 그리고 다음 달에도, 그 다음 달에도 같은 퀄리티.

---

## Phase 3: 도구를 연결한다

> GPT/Gemini와의 결정적 차이: **분석 → 공유 → 기록이 한 흐름**

### Step 5: Slack 연동

```
"방금 만든 글로우랩 리포트를 Slack #team-performance 채널에 보내줘"
```

### Step 6: Notion 연동

```
"3월 리포트를 Notion '클라이언트 리포트' 데이터베이스에 기록해줘.
클라이언트명, 월, KPI 달성 여부, 핵심 인사이트 3줄 요약으로"
```

### Step 7: 모닝 루틴 스킬로 확장

```
"매주 월요일 아침에 실행할 /ae-morning 스킬을 만들어줘.

동작:
1. 이번 주 클라이언트 미팅 일정 확인 (Google Calendar)
2. 미팅 예정 클라이언트의 최근 캠페인 성과 요약 (LEVER 데이터)
3. 지난주 Slack에서 클라이언트 관련 주요 메시지 정리
4. 오늘 해야 할 일 우선순위 제안
5. 결과를 Slack DM으로 나에게 보내기"
```

### 이 단계의 의미

GPT에서 리포트를 뽑으면? 텍스트를 복사 → Slack에 붙여넣기 → Notion에 수동 입력 → 캘린더 따로 확인. 여기서는 **대화 하나에서 분석-공유-기록-알림**이 전부 연결된다.

---

## Phase 4: 제안서 — "구조를 잡는 도구"

> 글은 사람이 쓴다. Claude Code는 **리서치와 구조**를 해준다.

제안서의 시간 분배: 자료 조사(40%) + 구조 잡기(20%) + 글쓰기(30%) + 디자인(10%).
Claude Code가 하는 것: 앞의 60%. 사람이 하는 것: 뒤의 40%.

### 상황

뷰티 브랜드 '블룸랩' RFP. 에코마케팅, 와이즈버즈와 경쟁PT.

### Step 8: 리서치를 병렬로

```
"블룸랩 제안서 준비를 위해 3가지를 동시에 조사해줘:

1. 클린뷰티 한국 시장 현황 (규모, 성장률, 주요 브랜드, 트렌드)
2. 에코마케팅과 와이즈버즈의 강점/약점 (competitor_benchmark + 웹 리서치)
3. Meta + Naver SA + TikTok 뷰티 업종 벤치마크 (CPC, CTR, CVR 범위)"
```

서브에이전트 3개가 병렬로 돌아간다. GPT에서는? 하나씩 순서대로 검색.

### Step 9: 구조를 잡는다

```
"리서치 결과를 종합해서 제안서 구조를 만들어줘.

목차 + 각 섹션별 핵심 메시지 + 슬라이드 키 카피.
15~20장 분량. PPT로 바로 옮길 수 있는 수준으로.

매드업 차별화 포인트: LEVER Xpert 기반 데이터 운영 역량"
```

### 이 단계의 의미

제안서의 최종 스토리텔링은 사람의 몫이다. 하지만 "시장이 어떤 상황이고, 경쟁사가 뭘 잘하고, 우리가 어떤 구조로 말할 것인가"의 **뼈대를 잡는 데 드는 시간**을 2주에서 반나절로 줄여준다. AE는 빈 PPT 앞에서 막막해하는 대신, 구조가 잡힌 상태에서 시작할 수 있다.

---

## Phase 5: 크리에이티브 브리프 — "감이 아니라 근거"

> 디자이너에게 "예쁘게 해주세요" 대신 "이 데이터 때문에 이 방향입니다"

### Step 10: 데이터 기반 브리프

```
"글로우랩의 3월 크리에이티브 데이터를 분석해서
4월 소재 크리에이티브 브리프를 작성해줘.

1. 먼저 소재 유형별(image/video/carousel),
   콘셉트별(UGC/전문가추천/시즌프로모) CTR/CVR 비교
2. 가장 효과적이었던 조합 기반으로 4월 방향 설정
3. 브리프 포맷:
   - 추천 소재 유형 + 근거 (3월 데이터 인용)
   - 카피 방향 3안
   - 사이즈/포맷 스펙
   - 수량
   - 일정 가이드"
```

### 이 단계의 의미

수정 요청이 90%인 이유: 처음 방향이 주관적이니까. "릴스가 요즘 대세라서 릴스로"가 아니라 **"3월 데이터에서 video+UGC 조합이 CTR 2.1%로 image+혜택강조(1.3%) 대비 62% 높았으므로"**라는 근거가 있으면, 광고주 수정 요청에도 방어가 된다.

## 에필로그

**오후 3시.**

오전에 워크스페이스를 세팅하고, 스킬을 만들고, 리포트를 돌리고, 제안서 구조를 잡고, 브리프를 보냈다.

내일 아침에는 `/ae-morning`을 실행하면 된다.
다음 달에는 `/monthly-report 글로우랩`을 실행하면 된다.
새 RFP가 오면 리서치 스킬을 돌리면 된다.

이건 "AI가 일을 대신 해준다"가 아니다.
**내 업무 방식을 내가 직접 설계한 것이다.**

LEVER가 광고 데이터를 자동화했듯이,
워크스페이스는 **AE의 나머지 모든 업무**를 자동화한다.
