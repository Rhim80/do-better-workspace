## Claude Code란?

### 일반 Claude와의 차이

| 구분 | 일반 Claude (웹/앱) | Claude Code |
|------|---------------------|-------------|
| 위치 | 브라우저에서 실행 | 내 컴퓨터에서 실행 |
| 역할 | 대화만 가능 | 실제 작업 수행 |
| 결과 | "이렇게 하세요" 조언 | "다 했습니다" 실행 |

**일반 Claude**: "발주 데이터를 분석하려면 이렇게 하세요..."

**Claude Code**: "발주 데이터를 분석했습니다. 차트를 만들고 보고서를 저장했어요. 주요 인사이트는..."

> 컨설턴트가 조언만 하는 것 vs 직접 실행까지 해주는 것의 차이

## 미션 1: 일반 데이터 분석

먼저 데이터를 있는 그대로 분석해본다. 프레임워크 없이, 자연어로 질문하는 방식.

### 분석 요청

```
"@royal/royal_orders_2602.csv 랑 @royal/royal_online_sales_2602.csv 랑 @royal/royal_inventory_data.csv
이 3개 파일 전부 종합해서 대표님 보고용 전채널 종합 보고서 만들어줘.

포맷은 이거야:
- 핵심 숫자 3개 (전채널 합산 매출, B2B vs 온라인 비중, 거래처유형별 매출)
- '그래서 뭐?'에 대한 답 3개 (왜 이런 숫자가 나왔는지 인사이트)
- 봄 이사철 액션 아이템 3개 (영업, 온라인, 재고/생산 각각)

차트도 넣어줘.
특히 BLOEI 비데 발주 추이랑 재고 위험 품목 표시해줘.
대표님이 비주얼 중요하게 보시거든요."
```

### 결과 확인 후 추가 질문

```
"거래처유형별 매출 비중 비교해줘. 시공사 대량 발주가 전체에서 얼마나 차지하는지"
```

```
"지역별 발주 분포 봐줘. 경기 신도시 물량이 얼마나 되는지"
```

```
"이 데이터 기준으로 봄 이사철 재고가 며칠 버틸 수 있는지 시뮬레이션해줘. 수요 30% 증가 가정하고"
```

### 포인트

> 이것이 Claude Code의 기본 능력입니다.
> 자연어로 물어보면 데이터를 읽고, 분석하고, 차트까지 만들어줍니다.
> 하지만 분석의 깊이는 질문하는 사람의 관점에 따라 달라집니다.

## 미션 2: 프레임워크 기반 분석 (@템플릿)

이번에는 **분석 프레임워크**를 함께 참조시킨다. 동일한 데이터에서 전혀 다른 깊이의 인사이트가 나오는 걸 보여준다.

### 왜 프레임워크인가?

| 일반 분석 | 프레임워크 분석 |
|-----------|----------------|
| "시공사 매출이 많다" | "상위 5개 거래처 HHI 3,200 = 고집중. 의존도 리스크 존재" |
| "CITY 수전이 잘 팔린다" | "CITY 수전 ABC 분석 A등급. 재고일수 1.5일로 긴급생산 필요" |
| "경기 지역이 많다" | "경기 매출 비중 40%. 단일 지역 의존도 40% 초과 = 영업 분산 필요" |

같은 숫자인데, 프레임워크가 **판단 기준**을 제공합니다.

### 분석 요청

```
"@royal/royal_orders_2602.csv 를 @b2b-account-framework.md 관점으로 분석해줘.

1. 거래처 집중도: 상위 5개 거래처 매출 비중, HHI 계산
2. 거래처 유형별 수익성: 대리점/시공사/인테리어업체/관공서 RFM 비교
3. 지역별 수요: 수도권 vs 비수도권, 지역 HHI
4. 발주 패턴: 주차별 발주 추이, OTIF 납기 준수율, 발주 취소 원인
5. 거래처 포트폴리오 매트릭스: Star/Cash Cow/Question Mark/Dog 분류

종합해서 대표님 보고 포맷으로 정리해줘."
```

### 결과 확인 후 추가 질문

```
"재고 데이터도 같이 봐줘. @royal/royal_inventory_data.csv 를 @inventory-analysis-framework.md 관점으로.
ABC 분석해서 A등급 품목 중 재고일수 7일 미만인 것 긴급 리스트 만들어줘"
```

```
"이 두 분석 결과를 합쳐서 @executive-report-framework.md 포맷으로 1페이지 보고서 만들어줘.
피라미드 원칙으로: 결론 1문장 + 근거 3개 + 액션 아이템 3개"
```

### 포인트

> 프레임워크는 AI에게 "어떤 관점으로 봐라"를 알려주는 렌즈입니다.
> 동일한 데이터도 어떤 프레임워크를 적용하느냐에 따라
> 완전히 다른 깊이의 인사이트가 나옵니다.
>
> 미션 1의 일반 분석은 "무엇이 보이는가"를 알려주고,
> 미션 2의 프레임워크 분석은 "무엇을 해야 하는가"를 알려줍니다.

## 데이터 분석 팁

Claude Code가 잘 분석하려면 데이터가 다음 조건을 갖추면 좋습니다:

- 날짜 형식: YYYY-MM-DD (예: 2026-02-01)
- 명확한 컬럼 헤더
- 일관된 데이터 형식
- 병합된 셀 없음

## 실제로 배운 것

이 실습을 마치면 여러분은 이미 다음을 익힌 것입니다:

- **데이터 분석**: 코드 없이 CSV에서 인사이트 도출
- **프레임워크 적용**: @템플릿으로 분석 깊이를 높이는 방법
- **보고서 자동화**: 여러 데이터를 종합한 경영진 보고서
- **의사결정 지원**: "그래서 뭐?"에 답하는 근거 기반 액션 아이템

> 코딩을 배우는 게 아닙니다. AI에게 위임하는 법을 배우는 것입니다.
