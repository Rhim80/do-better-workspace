# Setup Workspace - 워크스페이스 초기 설정

참여자의 업무 환경을 분석하여 도메인에 맞는 폴더 구조와 CLAUDE.md를 자동 생성합니다.

---

## Phase 1: 프로필 수집 (하나씩 질문)

### Q1. 이름

```
안녕하세요! 워크스페이스 초기 설정을 시작합니다.
이름이 어떻게 되세요?
```

### Q2. 회사/팀/역할

```
어떤 회사에서 어떤 일을 하고 계세요?
(예: "○○전자 마케팅팀 대리", "카페 사장", "프리랜서 디자이너")
```

### Q3. 주요 업무

```
매일 또는 매주 반복하는 업무가 있으면 알려주세요.
(예: "매출 보고서 작성", "인스타 콘텐츠 기획", "고객 CS 응대", "재고 발주")
```

### Q4. 진행 중인 프로젝트

```
지금 진행 중이거나 곧 시작할 프로젝트가 있으면 1~2개 알려주세요.
(예: "신메뉴 런칭", "홈페이지 리뉴얼", "없어요" 다 괜찮아요)
```

## Phase 2: 업무 폴더 스캔 (선택)

### Q5. 폴더 스캔 제안

```
혹시 바탕화면이나 문서 폴더에 업무 파일들이 있으세요?
있으면 어디쯤인지 대략 알려주세요. 폴더 구조를 참고해서 워크스페이스를 맞춤 설계해드릴게요.

예시:
- "바탕화면에 다 있어요"
- "문서 폴더에 업무별로 나눠져 있어요"
- "드라이브에 있어서 로컬에는 없어요"
- "잘 모르겠어요" → 이것도 괜찮아요, Phase 1 답변만으로 설계합니다
```

**경로 해석 규칙**:
- "바탕화면" → `~/Desktop`
- "문서 폴더" → `~/Documents`
- "다운로드" → `~/Downloads`
- 구체적 경로를 주면 그대로 사용
- "모르겠어요" / "없어요" → Phase 3로 바로 이동 (Phase 1 답변 기반)

### 폴더 스캔 실행 (경로가 있을 때)

```bash
# 1. 최상위 폴더 목록 (1-depth)
ls -1d <경로>/*/

# 2. 파일 확장자 분포
find <경로> -maxdepth 3 -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn | head -15

# 3. 상위 폴더별 하위 구조 (2-depth까지만)
find <경로> -maxdepth 2 -type d
```

**스캔 제한사항**:
- 최대 3레벨 depth
- 파일 내용은 절대 읽지 않음 (폴더명 + 파일명 + 확장자만)
- 숨김 폴더 (`.`으로 시작) 제외
- 결과가 100개 이상이면 상위 20개만 분석

### 스캔 결과 분석

스캔한 폴더명과 파일 패턴을 아래 매핑 테이블로 분류한다.

**10-projects (시한부, 종료 시점 있음)**:
프로젝트, 캠페인, 런칭, 리뉴얼, 이벤트, 프로모션, 개발, 구축, 기획안, 제안, 공모, launch, campaign, project, proposal

**20-operations (지속적, 반복적)**:
매출, 보고, 정산, 발주, 재고, 입고, 출고, 고객, CS, 인사, HR, 급여, 근태, 미팅, 회의, 일정, 점검, 체크리스트, 일보, 주보, 월보, 운영, 관리, daily, weekly, monthly, report, sales, inventory

**30-knowledge (축적, 참조용)**:
경쟁사, 트렌드, 리서치, 분석, 벤치마킹, 학습, 교육, 가이드, 매뉴얼, SOP, 레시피, 레퍼런스, 참고, 자료, research, reference, guide, analysis

**50-resources (외부 자료, 원본)**:
이미지, 사진, 로고, 소스, 원본, 템플릿, 양식, 서식, 다운로드, images, assets, templates, downloads

**분류가 애매한 폴더**:
해당 폴더에 대해 추가 질문한다.
```
"[폴더명]" 폴더는 어떤 성격의 자료인가요?
- 진행 중인 프로젝트 (끝나면 완료되는 일)
- 반복 업무 (계속 하는 일)
- 참고 자료 (쌓아두는 지식)
```

## Phase 3: 폴더 구조 설계 및 생성

### 3-1. 구조 제안

Phase 1~2에서 수집한 정보를 종합하여 폴더 구조를 제안한다.

**JD 번호 부여 규칙**:
- 10-projects: 11부터 순서대로 (11, 12, 13...)
- 20-operations: 21부터 순서대로 (21, 22, 23...)
- 30-knowledge: 31부터 순서대로 (31, 32, 33...)
- 50-resources: 51부터 순서대로 (51, 52, 53...)
- 폴더명은 한글 허용, 하이픈 구분: `21-매출보고`, `11-신메뉴-런칭`

**제안 형식**:

```
워크스페이스 구조를 이렇게 설계했어요:

10-projects/
  11-{프로젝트명}/     # {설명}
  12-{프로젝트명}/     # {설명}

20-operations/
  21-{운영업무명}/     # {설명}
  22-{운영업무명}/     # {설명}

30-knowledge/
  31-{지식영역}/       # {설명}

50-resources/
  51-{자료유형}/       # {설명}

(+ 기본 유지: 00-inbox, 00-system, 40-personal, 90-archive)

이대로 만들까요? 수정할 부분이 있으면 말씀해주세요.
```

### 3-2. 승인 후 폴더 생성

승인받으면:
1. 제안된 하위 폴더를 모두 생성 (`mkdir -p`)
2. 각 폴더에 빈 `.gitkeep` 파일 생성 (Git 추적용)

### 3-3. CLAUDE.md 생성

워크스페이스 루트에 `CLAUDE.md`를 생성한다.

**CLAUDE.md 템플릿**:

```markdown
# {회사명} 워크스페이스

## 사용자 프로필

**이름**: {이름}
**역할**: {회사} {팀/역할}
**주요 업무**: {Phase 1 Q3 답변 기반, 쉼표 구분}

## 폴더 구조

{Phase 3에서 승인된 구조를 여기에 그대로 반영}

## 업무 규칙

- 파일 수정 시 기존 파일 편집 우선 (새 파일 생성 최소화)
- 커밋은 요청 시에만
- Johnny Decimal 네이밍: `[숫자]-[설명적-이름]`
- {도메인 특화 규칙: Phase 1-2에서 파악한 업무 성격에 맞게 1-2개 추가}

## 주요 커맨드

| 커맨드 | 용도 |
|--------|------|
| `/daily-note` | 오늘 Daily Note 생성 |
| `/thinking-partner` | 생각 정리 파트너 |
| `/todo`, `/todos` | 할 일 관리 |
| `/idea` | 대화에서 아이디어 추출 및 저장 |

## 현재 진행 중

{Phase 1 Q4에서 언급한 프로젝트가 있으면 여기에 기록}
{없으면 이 섹션 제거}
```

**CLAUDE.md 작성 원칙**:
- 총 줄 수 80줄 이하
- 폴더 구조에 각 폴더의 한 줄 설명 포함
- 도메인 특화 규칙은 최대 2개 (너무 많으면 Claude가 무시함)
- "현재 진행 중" 섹션은 프로젝트가 있을 때만 포함

## Phase 4: 첫 액션

### 4-1. 첫 Daily Note 생성

- 경로: `40-personal/41-daily/{YYYY-MM}/{YYYY-MM-DD}.md`
- 템플릿: `00-system/01-templates/daily-note-template.md` 사용
- 월별 폴더가 없으면 자동 생성

### 4-2. 완료 안내

```
설정이 완료되었습니다!

생성된 파일:
- CLAUDE.md - Claude가 당신의 업무 맥락을 이해하는 핵심 파일
- {생성된 폴더 목록 요약}
- 40-personal/41-daily/{오늘날짜}.md - 첫 Daily Note

다음 단계:
1. `/daily-note` - 매일 하루 기록
2. `/thinking-partner` - 생각 정리가 필요할 때
3. 자유롭게 질문하세요 - Claude가 폴더 구조를 이미 이해하고 있습니다

CLAUDE.md는 살아있는 파일입니다.
업무가 바뀌거나 프로젝트가 추가되면 "CLAUDE.md 업데이트해줘"라고 말하세요.
```

## 주의사항

- **하나씩 질문**: 한 번에 여러 질문하지 않기
- **간단하게**: 복잡한 설명 없이 핵심만
- **폴더 스캔 시 파일 내용 절대 읽지 않음**: 폴더명과 파일명만 분석
- **제안 후 반드시 확인**: 폴더를 만들기 전에 항상 승인 받기
- **CLAUDE.md는 80줄 이하**: 짧고 핵심적으로
