# store-interview — 내 매장 인터뷰 (personal-context 자동 작성)

> **한 줄**: "personal-context.md 어떻게 채우는지 모르겠다" — AI가 사장님께 단계별로 묻고 답변을 정리해 자동으로 채워줌.
> **언제 씀**: 부트캠프 시작 시 본인 personal-context.md를 빈 양식부터 채울 때. zone-diagnosis·entry-verdict 돌리기 직전 단계.
> **출력**: `personal-context.md` (워크스페이스 root 또는 `participants/{이름}/personal-context.md`)

---

## 사장님 입장에서 보면

**이게 뭘 해주나요?**
양식 보고 막막한 사장님께 AI가 단계별로 짧게짧게 물어보고, 사장님이 답하시는 대로 personal-context.md에 자동으로 채워 넣어줍니다. 모르는 항목은 그냥 "모름"이라고 답하시면 됩니다 — 빈칸으로 두고 다음으로 넘어갑니다.

**이럴 때 쓰세요**
- "personal-context.md 양식 봤는데 어디부터 채워야 할지 모르겠다"
- "객단가·임대비중 같은 숫자를 어떻게 계산하는지 모르겠다"
- "페르소나 비중을 어떻게 잡아야 할지 감이 안 잡힌다"
- 부트캠프 첫날 11:30~12:30 본인 매장 정보 채우는 시간

**결과로 손에 쥐는 것**
- `personal-context.md` 자동 채워진 파일 (다 채워진 항목 + 빈칸 표시된 항목)
- 모르는 항목은 빈칸 + "현장 답사 시 확인" 같은 메모
- 다음 스텝 안내 (`zone-diagnosis` 또는 `entry-verdict` 호출)

**주의**
- 추정값이라도 적는 게 빈칸보다 낫습니다. 단 "추정값" 라벨 표시.
- 매출·임대 같은 민감 정보는 본인 디스크에만 저장. AI 학습 데이터로 안 들어감.
- 모르는 항목은 그냥 모른다고 답해도 됩니다. 분석 스킬이 빈칸을 알아서 우회합니다.

---

## 입력

- 빈 또는 부분 작성된 `personal-context.md` (워크스페이스 root 또는 참가자 폴더)
- 사장님과의 대화 (자유 답변 + 선택지 답변 혼합)

## 인터뷰 6단계

각 단계는 3-5개 질문 묶음. 한 번에 다 묻지 않음. 사장님 답변 받으면 즉시 파일 해당 섹션 업데이트.

### Step 1 — 본인·브랜드 (가벼운 시작)

**질문 묶음**:
1. 성함과 브랜드(매장) 이름 알려주세요.
2. 우리 매장 한 줄로 소개하면? (예: "청담 한식 회식 전문", "홍대 페어링 카페")
3. 사장님 역할은? (대표 / 매장 매니저 / 운영팀)
4. 사업자번호는 (선택, 모르거나 안 적고 싶으면 패스)

**파일 업데이트**: `## 본인·브랜드` 섹션

---

### Step 2 — 본점 결 (가장 기본 정보)

**질문 묶음**:
1. 본점 위치는 어디인가요? 주소 또는 동 단위로 말씀하셔도 됩니다.
2. 평수는 대략 얼마? (실평수 기준, 확실치 않으면 "약 30평" 정도로)
3. 좌석 수는 몇 석? (테이블 + 바 + 외부 합계)
4. 운영시간은? (평일 / 주말 따로)
5. 정기휴무는? (없으면 "없음")

**파일 업데이트**: `## 본점 (분석 기준 매장)` 표

**모르는 항목 처리**:
- 평수 모르면 → "공간 크기 어느 정도? (예: 작은 카페 / 중간 / 50평 이상 큰 매장)" 같이 결로 추정
- 좌석 모르면 → 테이블 개수 × 4명 추정 가이드

---

### Step 3 — 페르소나 (손님 결)

**질문 묶음** (대화 결로 풀어 묻기):
1. 평일 점심·저녁 / 주말에 어떤 손님이 많이 오시나요?
2. 직장인 / 학생 / 데이트·친구모임 / 가족·외식 / 외국인 관광객 / 거주민 단골 중 비중이 큰 결 1-3개?
3. 각 결의 비중 대략? (예: 직장인 60% / 가족 30% / 관광객 10%) — 모르면 그냥 "주로 직장인" 정도로
4. 주력 결의 평균 객단가? (예: "직장인 점심은 1.2만원, 회식은 5만원")

**파일 업데이트**: `## 페르소나 (주요 손님 결)` 표

**추정 가이드**:
- 객단가 모르면 → "한 달 매출 / 한 달 손님 수" (POS 통계 보면 됨)
- 또는 "평일 하루 매출 / 평일 하루 손님 수"
- 비중 모르면 → 평일 vs 주말 / 점심 vs 저녁 비중부터 시작

---

### Step 4 — 메뉴·카테고리

**질문 묶음**:
1. 카테고리는? (한식 / 한식·구이 / 카페 / 분식 / 중식 / 일식 / 양식 / 다이닝 / 바 / 기타) — 가장 가까운 1개
2. 시그니처 메뉴 3개?
3. 객단가 분포: 가장 싼 주문 / 평균 / 가장 비싼 주문

**파일 업데이트**: `## 메뉴·카테고리` 섹션

**카테고리 매핑**:
- 헷갈릴 때 → 매출 비중 가장 큰 메뉴 결로 결정
- 카페+베이커리는 "카페", 카페+다이닝은 "다이닝"

---

### Step 5 — 운영비 구조 (가장 어렵지만 중요)

**질문 묶음** (한 번에 묻지 말고 하나씩):
1. 월매출 대략? (확실한 수치 X도 OK, "약 3천~4천", "월 5천 정도")
2. 월 임대료? (보증금도 같이)
3. 인건비 대략? (정직원 + 파트타임 합계)
4. 원자재 매출 대비 비중? (예: 30%) — 모르면 패스
5. 영업이익 대략 또는 흑자/적자 여부

**자동 계산 추가**:
- 임대비중 = 임대료 / 월매출 × 100 (자동)
- rent_ratio_max 추천:
  - 카페·테이크아웃: 8%
  - 한식·일반음식: 10%
  - 다이닝·바: 12%

**파일 업데이트**: `## 운영비 구조` 표 + `rent_ratio_max`

**민감 정보 안내**:
- "이 정보는 본인 디스크에만 저장됩니다. AI 학습 데이터로 안 들어가요. 단 시연 화면 공유하실 때는 표시 여부 확인."
- 정말 적기 싫으면 "비공개" 또는 빈칸 OK

---

### Step 6 — 신규 출점 + 현재 매장 고민

**질문 묶음 A — 출점**:
1. 2호점 또는 신규점 검토 중이신가요? (예/아니오)
2. (예 시) 검토 후보 상권 1-3곳? (대략 동 단위)
3. 검토 평수 대략?
4. 본점과 거리가 가까우면 카니발(잠식) 우려는?

**질문 묶음 B — 현재 매장 고민**:
1. 최근 12개월 매출 추세? (증가/유지/감소)
2. 매장 앞 유동인구 체감? (증가/유지/감소)
3. 외국인 관광객 비중 대략?
4. SNS·블로그 검색 관심도? (브랜드 후기, 인스타 태그 등)
5. **지금 가장 큰 고민 한 줄**

**파일 업데이트**: `## 신규 출점 고민` + `## 현재 매장 고민` 섹션

---

## 단계 완료 후 — 점검 + 다음 스텝 안내

### 점검 (핵심 항목 채워졌는지)

- [ ] 본점 위치(행정동)
- [ ] 카테고리
- [ ] 평균 객단가
- [ ] 운영시간
- [ ] 검토 후보 상권 1개 이상 (출점 고민이 있는 경우)

위 5개가 채워지면 분석 가능. 빠진 항목 있으면 한 번 더 보강 질문.

### 다음 스텝 안내

매장 결에 따라 자동 추천:
- **기존 매장 매출이 빠지고 있다** → `zone-diagnosis {본점_zone}` 추천
- **2호점 검토 중** → `entry-verdict {후보_zone}` 추천 (3곳이면 비교 매트릭스)
- **둘 다** → 진단 먼저, 그 다음 후보 비교

---

## 인터뷰 톤 가이드

- **짧게 묻기**: 한 묶음 3-5개 질문 이내
- **선택지 + 자유 답변 혼합**: 막막할 만한 질문은 "예: ___, ___ 중 가장 가까운 것"
- **모름 허용**: "확실치 않으면 그냥 '모름'이라고 답하셔도 됩니다"
- **추정 가이드 제공**: 객단가·임대비중 등 계산식 함께
- **압박 X**: "민감하면 비공개로 두셔도 됩니다"
- **사장님 결 존중**: F&B 도메인 용어 친근하게 (회전율, 객단가, 카니발, BEP 등은 풀어서)

## 출력 형식

`personal-context.md`를 양식 그대로 유지하되:
- 채워진 항목: 그대로
- 빈 항목: 빈칸 (또는 "(미확인)" 표시)
- 추정값: 값 옆에 "(추정)" 라벨
- 민감 비공개: "(비공개)"
- 모르는 항목: "(현장 답사 시 확인)"

## 체크리스트

- 빈 personal-context.md 양식 존재 확인 (없으면 워크스페이스 root에서 복사)
- 인터뷰 결과 즉시 파일 업데이트 (Edit 또는 Write)
- 핵심 5개 항목 채워졌는지 마지막 점검
- 다음 스텝(zone-diagnosis 또는 entry-verdict) 추천

## 주의

- 한 번에 다 묻지 말 것. 6단계 분할 필수.
- 모르는 항목 강요 X. 빈칸 OK.
- 매출·임대 등 민감 정보는 본인 동의 받고 적기.
- 화면 공유 중인 경우 민감 정보는 별도 안내 후 작성.
