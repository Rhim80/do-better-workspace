# zone-diagnosis 한 페이지 카드 템플릿

> zone-diagnosis Step 7에서 사용. zone-report로 HTML/PDF 변환 가능.

```
┌─────────────────────────────────────────────────────────┐
│ Diagnosis · {zone} {category} ({brand})                  │
│ 진단일 {date}                                              │
├─────────────────────────────────────────────────────────┤
│ 4분면 신호: {quadrant_emoji} {quadrant_label}             │
│   생활인구 {living_pop_change} ({living_pop_trend})        │
│   매출      {sales_change} ({sales_trend})                 │
├─────────────────────────────────────────────────────────┤
│ 카테고리 변화율 (최근 4Q vs 이전 4Q)                       │
│   {my_category}      {my_change} {my_arrow}              │
│   {adjacent_1}       {adj1_change}                       │
│   {adjacent_2}       {adj2_change}                       │
│   {adjacent_3}       {adj3_change}                       │
│   {adjacent_4}       {adj4_change}                       │
│   {adjacent_5}       {adj5_change}                       │
├─────────────────────────────────────────────────────────┤
│ 외국인 방문: {foreign_change} ({foreign_trend_label})      │
│ 검색 관심도: "{search_keyword}" {search_change}            │
├─────────────────────────────────────────────────────────┤
│ 가설 3개:                                                 │
│   A. {hypothesis_a}                                       │
│      근거: {evidence_a}                                   │
│      신뢰: {confidence_a}                                 │
│   B. {hypothesis_b}                                       │
│      근거: {evidence_b}                                   │
│      신뢰: {confidence_b}                                 │
│   C. {hypothesis_c}                                       │
│      근거: {evidence_c}                                   │
│      신뢰: {confidence_c}                                 │
├─────────────────────────────────────────────────────────┤
│ 다음 액션: {next_action}                                  │
│   (예: entry-verdict로 다른 상권 점프 / 본인 매장 차별화)  │
├─────────────────────────────────────────────────────────┤
│ Sources: {sources_summary}                                │
│ 가설은 데이터 패턴. 인과는 현장 답사·인터뷰로 검증.          │
└─────────────────────────────────────────────────────────┘
```

## 가설 후보 라이브러리

진행자가 신호 조합 보고 채택:

| 신호 패턴 | 가설 후보 |
|-----------|-----------|
| Q2 + 외국인 빠짐 | "외국인 의존 카테고리(명품·뷰티) 빠지며 동반 영향" |
| Q2 + 명품·패션 빠짐 | "비소비형 체류 (SNS·산책) 진행" |
| Q2 + 검색 빠짐 선행 | "카테고리 자체 매력도 약화 (트렌드 이탈)" |
| Q3 + 인접 카테고리 빠짐 | "상권 자체 위축, 출구 모드" |
| Q3 + 본인만 빠짐 | "컨셉·운영 이슈 (우리 매장 문제)" |
| Q4 + 객단가 상승 | "충성고객 강화 / 인플레이션 흡수" |
| Q1 + 임대 상승률 > 매출 | "성장하지만 BEP 악화, 임대 협상 필요" |

## 신뢰도 판정

- **높음**: 3개+ 데이터 소스에서 같은 방향 신호
- **중**: 2개 소스 일치 또는 1개 소스 강한 신호
- **낮음**: 단일 소스 또는 데이터 결측 다수
